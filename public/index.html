Built for the agent era

Your software assumed a human user. AI agents don't click dashboards. They call APIs. They act. They decide.

When a human approved an action in a workflow, they were simultaneously executing it, reviewing it, and accepting responsibility. The human in the loop was the audit function.

Remove the human. Something has to replace the audit function. DarkMatter records those decisions — automatically, at API level, with zero dashboard interaction required. One endpoint. Any agent. Any language.

How it works

Three things. No monitoring, no model lock-in, no workflow replacement.

DarkMatter records what your agents did, lets you inspect and replay it, and lets you prove it to anyone who wasn't there.

dm.commit()

Record every decision as it happens

Every agent action becomes a context commit — input, output, model, tool calls — linked to the action before it, stored outside your system the moment it's made.

dm.replay() / dm.fork()

Walk back to any point in the chain

Replay the full decision chain step by step. Branch from any checkpoint to try a different model or prompt without touching the original run. The original is always intact.

dm.export() / dm.verify()

Prove it to anyone — without asking anyone

Export a self-contained proof bundle anyone can verify — no DarkMatter account, no internet connection, no asking you to vouch. The record proves itself.

Proof levels

Not every record carries the same weight. DarkMatter makes that visible.

Start with Recorded — no code changes. Move to Signed or Sealed as your compliance requirements grow.

Level What it means How you get it
Recorded DarkMatter captured this record. You can inspect it, but some trust in the capture method is still required. Proxy or browser extension — no code changes needed.
Signed The agent signed this record before it reached DarkMatter. It cannot have been changed in transit without detection. SDK — one extra line when your agent commits.
Sealed Signed and encrypted before leaving your system. DarkMatter never saw the content. Zero-knowledge verification. SDK with your own encryption key (BYOK).
